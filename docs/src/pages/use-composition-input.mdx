# useCompositionInput

import ExportMetaInfo from '../components/export-meta-info';

<ExportMetaInfo />

Track value of **uncontrolled** `<input />`, but optimized for CJK (Chinese, Japanese, Korean, or other non-latin langauges) IME. `useCompositionInput` accepts only only one parameter `onChange`, which will only be triggered when the composition is finished and the value is actually emitted.

The `useCompositionInput` is **uncontrolled** because the `<input />`'s `value` attribute needs to reflect the current composition state, while the "real value" should not include compositions. Also, due to the very nature of the CJK IME, the re-rendering count will be a lot more than the user's actual keystroke count, which can make your React less responsive.

## Usage

```tsx copy
import { useCompositionInput } from 'foxact/use-composition-input';
import { useCallback } from 'react';

const Example = () => {
  const inputProps = useCompositionInput(useCallback((value: string) => {
    // Do something with the value
  }, []));

  return (
    <input
      {...inputProps}
      // useCompositionInput is uncontrolled, so you might need to provide defaultValue
      defaultValue={defaultValue}
    />
  );
}
```
